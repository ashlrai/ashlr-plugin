---
title: MCP tools
description: 27 token-efficient drop-in replacements for Claude Code's built-in tools, plus 13 ashlr-only utilities.
---

ashlr ships 40 MCP tools. The bulk of them are drop-in replacements for Claude Code's built-ins — `Read`, `Grep`, `Edit`, `Bash`, etc. — that compress, summarize, or RAG-route to use fewer tokens. The rest are utilities that don't have built-in equivalents (genome management, savings reporting, GitHub write ops, semantic diffing, etc.).

Every tool's name is prefixed `ashlr__` so it never collides with a built-in.

## Reads & search

| | |
|---|---|
| [`ashlr__read`](/docs/tools/read) | snipCompact head + tail; middle elided. The single biggest token saver. |
| [`ashlr__grep`](/docs/tools/grep) | Genome-RAG-backed; ~84 % savings on warm queries. |
| [`ashlr__glob`](/docs/tools/glob) | Globbing with sane head + tail elision on giant result sets. |
| [`ashlr__tree`](/docs/tools/tree) | Per-dir truncation visible to the model. |
| [`ashlr__ask`](/docs/tools/ask) | Codebase Q&A backed by the genome. |
| [`ashlr__orient`](/docs/tools/orient) | One-shot project orientation snapshot. |

## Cost-savings methodology

Every tool's per-call savings is measured against the equivalent Claude Code built-in on a fixed corpus of files ≥ 2 KB. See [Concepts → Savings math](/docs/concepts/savings-math) for how the numbers in the per-tool tables are computed.
