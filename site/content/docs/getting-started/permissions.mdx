Claude Code prompts for permission the first time each MCP tool is called. For 15 tools that means 15 interrupts on first use. `/ashlr-allow` writes an allowlist so the prompts never appear.

## What it does

Running `/ashlr-allow` invokes `scripts/install-permissions.ts`, which:

1. Reads `.claude/settings.json` (or creates it).
2. Appends each ashlr tool name to the `allowedTools` array.
3. Writes the file back.

## Scope

`/ashlr-allow` writes to the **project-level** `.claude/settings.json` by default. To add them globally (all projects), pass `--global`:

```sh
bun run ~/.ashlr-plugin/scripts/install-permissions.ts --global
```

## Dry run

Preview what would be written without writing:

```sh
bun run ~/.ashlr-plugin/scripts/install-permissions.ts --dry-run
```

## Undo

```sh
bun run ~/.ashlr-plugin/scripts/install-permissions.ts --remove
```

This removes only the ashlr entries and leaves the rest of `settings.json` intact.

## Reload required

After any settings change, restart Claude Code or run `/reload-plugins` before the new permissions take effect.
